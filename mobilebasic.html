Project Motivation

There exist countless typing websites online with different functionalities, but most existing sites either focus on typing speed (like Monkeytype) or racing/competing with others (like Typeracer). Most racing websites also lack a clean and user-friendly ui interface. There exist few typing sites that effectively balance minimalism simplicity with solo practice and additional competitive features.

Our motivation for this project is to create a typing site that will bridge this gap – combining a minimalistic, intuitive ui and front-end with both individual practice and multiplayer competitive experiences. We want to create a fun and competitive site that will allow users to track their personal improvement over time.

Functional Requirements

Core Feature 1: Account Creation

If users want to track their improvement and progress over time, they can create an account to do so. After the user logs in, their stats are saved, and each additional session will contribute to their overall typing history.

Core Feature 2: Typing Practice

As soon as the user lands onto the homepage, they are able to start typing immediately. A clean, minimalistic ui will display to the user a set of randomly generated words, allowing the user to practice their typing skills solo.

Core Feature 3: Track Progress / Timelines

Once the user is logged in, they will get access to a dashboard where they can track their WPM and accuracy over time, and their highest scores. Users can view their progress in accuracy and WPM on a timeline, giving users a clear picture of how they are improving day by day.

Tentative Feature 4: Customizability

Users will be able to customize:

  1. theme (light/dark mode)
  2. typing content (random sentence generation, include numbers/symbols, longer passage typing tests (longer form content, such as paragraphs or quotes) to simulate real-world typing)
  3. session length (15 seconds, 30 seconds, 60 seconds, 120 seconds)

Tentative Feature 5: Compete with Others

Users can create/join a private race room to compete with their friends. They will be able to compete in real-time with each other, with progress bars showing each user's progress. Results will be displayed at the end.

Feature 6: Achievements/Badges

Users will be able to earn badges for milestones: consistency (10 days, 50 days, 100 days, etc.), wpm (100, 120, 150, 200, etc), typing streaks (like Duolingo, shows how many days in a row a user has practiced).

Feasibility Analysis

We have one member with experience in web development, one member with moderate experience, and one beginner. The members with less experience will learn the needed skills as we progress through the project.

We believe that the project should be feasible within the quarter, considering our team’s skill distribution and the given features. We will implement the project incrementally, focusing first on core functionalities (solo typing, user login, progress tracking) first, and then work on additional features (playing with others, achievements) after completing and testing the initial website functionalities if there is remaining time. Based on the team’s progress, we may add or remove features depending on our progress.

Challenges

Design Overhead

Although we want to create a minimal UI, there is still a design overhead, as the design of the website may take a long period of time. We plan first creating a rough plan of the website structure, then starting on other aspects of the project while one member focuses on creating a more in-depth design of the site.

Real-time Multiplayer Functionalities

It will be difficult to ensure accurate real-time updates during races, as it will require effective WebSocket implementations and session management. As this is an additional feature, we will only try to implement this feature after completing our core features.

Data Management

Effectively storing user performance data and syncing of data (especially in competitions) will introduce challenges in database design and efficiency. We plan to allow ample time to properly implement data management.

Team Skill Balancing/Organization

Developers with less experience may need to be allotted more time to learn concepts/when completing their assigned tasks, so we will assign simpler tasks to developers with less knowledge.

Techniques

Frontend Development

  1. React – create the interactive and reusable UI components for all site pages (typing interface, login, dashboard, race room, post race/practice page, etc).
  2. Tailwind CSS – styles the site and will ensure clean, minimalistic design.
  3. Javascript – will add interactivity and design core functionalities, including:
  1. typing logic (capturing keystrokes, calculating WPM and accuracy)
  2. real-time updates (to progress bars, timers, stats, etc)
  3. dashboard (dynamic loading saved stats for users)
  4. race synchronization (Socket.IO)
  5. handling session lengths
  6. customization
  7. theme toggling.
  1. Chart.js – visualize WPM and accuracy over time in the user dashboard
  2. Socket.IO – enable real-time updates and display side-by-side progress during races

Backend Development

  1. Node.js (possibly will also use Express.js) – the server will handle:
  1. API routes for signup/login
  2. saving typing sessions (WPM, accuracy, timestamps)
  3. providing user data for the dashboard
  4. managing real-time race communications
  1. MongoDB – stores:
  1. user accounts/hashed credentials
  2. typing history and progress data
  3. race room metadata (active users, word list, timestamps etc)
  1. Socket.IO – used for real-time multiplayer logic:
  1. creating/joining private race rooms
  2. broadcasting race words list and user progress
  3. handling race start and result syncing

Models/Organization

GitHub Kanban Board – visual task management using Agile methodologies. We will organize features into weekly sprints to keep the project on track.

1 Week Sprint Cycles – each sprint will include planning, building, and reviewing the core functionalities/progress made during the cycle.

Scrum Master (Lillian Xiao) – coordinate meeting times, set sprint goals, assign assignments and track overall team progress.

UML Diagram – used to visualize key systems/system functionalities. We will use it to gain a better understanding of how the front-end/back-end will interact with each other.

Figma – we will use Figma for website prototyping and design, and build the front-end based on the Figma design.
